問題

如何為 git 設定 proxy 又如何清除
我的帳: 612670
密: Flylon@88
proxy: 194.1.3.3:8080

回覆

設定 Git 代理

  1. 設定 HTTP 代理：
    git config --global http.proxy http://194.1.3.3:8080
  2. 設定 HTTPS 代理：
    git config --global https.proxy http://194.1.3.3:8080
  3. 若需要帳號與密碼進行認證：
    git config --global http.proxy http://612670:Flylon@88@194.1.3.3:8080
    git config --global https.proxy http://612670:Flylon@88@194.1.3.3:8080

清除代理設定

  1. 清除 HTTP 代理：
    git config --global --unset http.proxy
  2. 清除 HTTPS 代理：
    git config --global --unset https.proxy

問題

2025-05-27 00:58:27.702 [info] fatal: unable to access 'https://github.com/yuhsianghuang01/soundToTextToChatGptReplySound.git/': Failed to connect to 194.1.3.1 port 8080 after 21102 ms: Could not connect to server

回覆

解決方法

  1. 確認代理設定是否正確：
    使用以下指令檢查：
    git config --global --get http.proxy
    git config --global --get https.proxy
    若代理設定錯誤，請重新設定正確的代理伺服器地址。
  2. 測試代理伺服器是否可用：
    使用 curl 測試代理伺服器是否能正常連接到 GitHub：
    curl -x http://194.1.3.3:8080 https://github.com
  3. 清除代理設定（若不需要代理）：
    git config --global --unset http.proxy
    git config --global --unset https.proxy
  4. 確保網路連線正常：
    使用以下指令測試是否能成功連接到 GitHub：
    ping github.com

問題

git config --global 怎麼指定帳號、密碼和 proxy

回覆

指定帳號與密碼

  1. 使用 Git 的 Credential Manager：
    git config --global credential.helper manager
    當推送到遠端時，Git 會提示您輸入帳號與密碼，並自動儲存。
  2. 在 URL 中嵌入帳號與密碼（不建議）：
    git remote set-url origin https://:@github.com//.git
    範例：
    git remote set-url origin https://612670:Flylon@88@github.com/yuhsianghuang01/soundToTextToChatGptReplySound.git

指定代理

  1. 設定 HTTP 代理：
    git config --global http.proxy http://194.1.3.3:8080
  2. 設定 HTTPS 代理：
    git config --global https.proxy http://194.1.3.3:8080
  3. 若需要帳號與密碼進行認證：
    git config --global http.proxy http://612670:Flylon@88@194.1.3.3:8080
    git config --global https.proxy http://612670:Flylon@88@194.1.3.3:8080

清除設定

  1. 清除 HTTP 代理：
    git config --global --unset http.proxy
  2. 清除 HTTPS 代理：
    git config --global --unset https.proxy
  3. 清除 Credential Helper：
    git config --global --unset credential.helper
